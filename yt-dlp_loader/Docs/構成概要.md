# 構成概要

## 1. 目的

`yt-dlp_loader` は、`yt-dlp` を毎回手打ちせずに実行するための小さな GUI ラッパーです。  
設定保存、URL 一括投入、cookie 利用、出力名ルール調整を 1 つの画面で扱うことを目的にしています。

## 2. 全体構成

このアプリは役割ごとに次のように分かれます。

- `Program.cs`
  アプリ起動入口です。`AppCompositionRoot` 経由で `Form1` を起動します。
- `AppCompositionRoot.cs`
  共有サービスを 1 か所で組み立てます。
- `Form1.cs`
  画面から値を集め、表示を更新します。
- `FormActionRunner.cs`
  画面イベントの例外処理とボタンの有効/無効切り替えを担当します。
- `ApplicationService.cs`
  画面から呼ぶ保存、実行、ブラウザ起動、フォルダ起動の入口です。
- `AppRuntimePaths.cs`
  `%LOCALAPPDATA%\\yt-dlp_loader` 配下の設定ファイルと config ファイルの場所をまとめます。
- `AppSettingsValidator.cs`
  実行前に必須設定と `yt-dlp.exe` の存在を確認します。
- `ProcessLauncher.cs`
  `yt-dlp`、ブラウザ、更新、フォルダ起動の共通プロセス起動を担当します。
- `BrowserUrlBatchOpener.cs`
  URL の順次ブラウザ起動と待機を非同期で扱います。
- `FormSettingsMapper.cs`
  画面コントロールと `AppSettings` の相互変換を担当します。
- `ConsoleTextBoxWriter.cs` / `ConfigFileTextBoxViewer.cs` / `UrlListBoxAdapter.cs`
  実行ログ表示、config 表示、URL リスト操作などの UI 補助を担当します。
- `UrlDragDropReader.cs`
  ドラッグ&ドロップされたデータから URL 文字列を取り出します。
- `AppSettingsStore.cs`
  `app-settings.json` の読込と保存を担当します。
- `YtDlpRunner.cs`
  保存、設定ファイル生成、実行の流れをまとめます。
- `YtDlpService.cs`
  `yt-dlp.conf` と URL ファイルを作り、`yt-dlp.exe` を起動します。
- `YtDlpConfigBuilder.cs`
  `yt-dlp.conf` に書くオプション行を組み立てます。
- `BrowserProfileRepository.cs`
  `browser_profiles.json` を読み、プロファイル解決を担当します。
- `BrowserExecutableResolver.cs`
  ブラウザ実行ファイルを候補パスまたは設定値から解決します。
- `BrowserCookieOptionBuilder.cs` / `BrowserLauncher.cs`
  cookie 指定生成とブラウザ起動を担当します。

## 3. 実行時に使うファイル

- `browser_profiles.json`
  ブラウザ表示名、ブラウザ種別、プロファイル名、プロファイルディレクトリを保持します。
- `%LOCALAPPDATA%\\yt-dlp_loader\\app-settings.json`
  画面で扱うユーザー設定を保存する正規の設定ファイルです。
- `%LOCALAPPDATA%\\yt-dlp_loader\\yt-dlp.conf`
  実行時に毎回生成される `yt-dlp` 用設定ファイルです。

## 4. 主な処理フロー

### 起動時

1. `Form1` がユーザー設定を読み込みます。
2. `FormSettingsMapper` が設定を画面へ反映します。
3. `app-settings.json` が無ければ旧 `user.config` を探して移行します。
4. ブラウザ候補を `browser_profiles.json` から読み込みます。
5. `browser_profiles.json` の問題があれば起動時に警告します。
6. `ConfigFileTextBoxViewer` が既存の `yt-dlp.conf` を画面に表示します。

### Save 時

1. `FormSettingsMapper` が画面の値を `AppSettings` にまとめます。
2. ユーザー設定を保存します。
3. `AppSettings` から `YtDlpOptions` を組み立てます。
4. cookie 指定を含む `yt-dlp.conf` を再生成します。

### Start 時

1. Save 相当の処理を行います。
2. `UrlListBoxAdapter` が ListBox の URL 一覧を取り出します。
3. ListBox の URL をファイルへ書き出します。
4. `--config-location` を付けて `yt-dlp.exe` を起動します。
5. 必要なら `BrowserUrlBatchOpener` が URL を順に開きます。
6. `ConsoleTextBoxWriter` がコンソール出力を画面へ追記します。

## 5. 命名ルールの組み立て

出力ファイル名は次の要素を順に連結して作られます。

1. 任意の先頭文字列
2. 動画タイトル
3. uploader 名の付加
4. video id の付加
5. 任意の末尾文字列 1
6. 任意の末尾文字列 2

最終的に `DL Folder` 配下へ `%(ext)s` 付きで保存されます。

## 6. 設計上の特徴

- GUI と実処理は `Form1`、`YtDlpRunner`、`YtDlpService` に分かれています。
- 画面と設定モデルの変換は `FormSettingsMapper` に集約しています。
- 設定ファイルと config ファイルの配置は `AppRuntimePaths` に集約しています。
- 実行前の必須入力確認は `AppSettingsValidator` に集約しています。
- 外部プロセス起動は `ProcessLauncher` に集約しています。
- 画面イベントの例外処理は `FormActionRunner` に集約しています。
- `yt-dlp.conf` の行組み立ては `YtDlpConfigBuilder` に集約しています。
- 実行ログ表示、config 表示、URL リスト操作は小さな UI 補助クラスへ分けています。
- URL の順次ブラウザ起動は非同期で行い、UI 停止を避けています。
- cookie の扱いとブラウザ起動は専用クラスへ分けてあり、画面側は表示名だけを扱います。
- `yt-dlp` への詳細指定は `yt-dlp.conf` に寄せ、起動引数は最小限にしています。

## 7. 現状の制約

- 実行入口は WinForms 画面中心で、CLI や IPC 入口はありません。
- ブラウザ実行ファイルは候補パス探索または `browser_profiles.json` の `executablePath` 前提です。
- `browser_profiles.json` が実行ファイル基準で配置されている前提です。
- 進捗管理や結果管理は簡易ログ表示が中心です。
