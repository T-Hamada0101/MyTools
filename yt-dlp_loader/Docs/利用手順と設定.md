# 利用手順と設定

## 1. 事前準備

次を用意します。

- `yt-dlp.exe`
- ダウンロード先フォルダ
- URL 一覧を書き出す先ファイル
- 必要なら `browser_profiles.json` に登録したブラウザプロファイル

## 2. 基本的な使い方

1. `yt-dlp.exe` に `yt-dlp` 本体のパスを設定します。
2. `url File` に URL 一覧の保存先を設定します。
3. `DL Folder` に保存先フォルダを設定します。
4. 必要ならブラウザプロファイル、ファイル名ルール、720p 制限を設定します。
5. URL をリストへ追加します。
6. `Start` を押して実行します。

## 3. 設定項目の意味

- `URLを既定のブラウザで開く`
  実行時に URL を順に開きます。
- `閉じるまでの秒数`
  URL を連続で開く間隔です。
- `DLスレッド数`
  `yt-dlp.conf` の `-N` として反映されます。
- `DLファイル名にuploaderを付加`
  `%(uploader)s` を付けます。
- `DLファイル名にidを付加`
  `%(id)s` を付けます。
- `DLファイル名の最初に次の文字を付加`
  タイトルの先頭へ固定文字列を付けます。
- `DLファイル名の最後に次の文字を付加`
  タイトルの末尾へ固定文字列を付けます。
- `720p以下に制限`
  `--format-sort res:720` を設定へ追加します。

## 4. 生成される設定

`Save` または `Start` を行うと、次の場所に画面設定と `yt-dlp.conf` が保存されます。

```text
%LOCALAPPDATA%\yt-dlp_loader\app-settings.json
```

```text
%LOCALAPPDATA%\yt-dlp_loader\yt-dlp.conf
```

`app-settings.json` は、画面の入力値と `prefix` / `suffix` のチェック状態を保持する正規の設定ファイルです。  
旧 `user.config` が見つかった場合は、初回読込時にこの JSON へ移行します。

この設定には主に次が出力されます。

- 出力テンプレート `-o`
- `--no-mtime`
- `--console-title`
- `-N`
- `--cookies-from-browser`
- URL 一覧ファイルを読むための `-a`
- 必要なら `--format-sort res:720`

## 5. browser_profiles.json の役割

`browser_profiles.json` は、画面のドロップダウン候補と cookie 指定の元データです。  
このファイルは実行ファイルと同じフォルダから読み込みます。  
各要素は次の意味を持ちます。

- `displayName`
  画面に表示する名前
- `browser`
  `firefox`、`chrome`、`edge` を指定します
- `profileName`
  ブラウザ起動時に使うプロファイル名
- `profileDir`
  `yt-dlp` の `--cookies-from-browser` に渡すプロファイルディレクトリ名
- `executablePath`
  任意です。ブラウザ実行ファイルの場所を明示したい時だけ指定します。

## 6. 更新機能

`Update` ボタンは、設定済みの `yt-dlp.exe` に対して `-U` を付けて管理者権限で実行します。  
`yt-dlp` の自己更新に依存するため、実行環境側で更新可能であることが前提です。

## 7. 注意点

- `url File` や `DL Folder` が空だと、意図した動作にならない可能性があります。
- `DLスレッド数` は `yt-dlp` の `-N` に反映されます。
- Chrome、Firefox、Edge は標準的なインストール先を探します。見つからない場合は `browser_profiles.json` の `executablePath` を使えます。
- `browser_profiles.json` が無い場合でも動きますが、cookie 周りは既定値ベースになります。
- `browser_profiles.json` が壊れている、または有効な定義が無い場合は、起動時に警告を表示します。
